# Exposition aux risques

L'outil `exposition_risques` évalue le niveau d'exposition aux risques majeurs pour des coordonnées données. Il retourne un **texte descriptif** pour chaque risque évalué.

## Parametres d'entrée

| Parametre | Type | Obligatoire | Description |
|---|---|---|---|
| `longitude` | `number` | Oui | Longitude (EPSG:4326, entre -180 et 180) |
| `latitude` | `number` | Oui | Latitude (EPSG:4326, entre -90 et 90) |
| `risques` | `string[]` | Non | Liste des codes de risques a évaluer (défaut: tous) |

## Codes de risques disponibles

| Code | Risque |
|---|---|
| `argiles` | Retrait-gonflement des argiles |
| `mouvement_terrain` | Mouvements de terrain |
| `cavites` | Cavités souterraines |
| `inondations` | Inondations (TRI, AZI, PAPI, PPRN) |
| `catnat` | Catastrophes naturelles |
| `icpe` | Installations classées Seveso |
| `installations_nucleaires` | Installations nucléaires |

## Exemple d'appel

```json
{
  "longitude": 2.3522,
  "latitude": 48.8566,
  "risques": ["argiles", "inondations"]
}
```

## Format de réponse

La réponse est un **texte** concatenant les resultats de chaque risque évalué. Par exemple :

```
Le niveau d'exposition au risque de retrait-gonflement des argiles
a l'adresse indiquee est : Exposition moyenne (score de 2 sur 3)

3 zones à risque d'inondation recensées dans un rayon de 5 kilomètres
autour de l'adresse indiquee. Voici la liste des zones et des risques
concernes :
  - TRI Paris exposée aux risques Debordement de cours d'eau
```

:::tip
Cet outil est idéal quand l'agent IA a besoin d'un résumé textuel des risques pour formuler une réponse à l'utilisateur. Pour une visualisation cartographique, utilisez plutôt [**carte_exposition_risques**](./carte-exposition-risques).
:::

## Rayon de recherche

La plupart des risques sont évalués dans un **rayon de 5 km** autour des coordonnées fournies, à l’exception de :
- **Argiles** : évaluation ponctuelle (pas de rayon)
- **Installations nucléaires** : rayon défini par l'API Géorisques
